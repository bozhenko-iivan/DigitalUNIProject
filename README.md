A console-based information system for managing university entities (Students, Teachers, Faculties, and Departments). Developed for the Advanced Java course at NaUKMA.

Our system has following hierarchy:

<pre>
  
								University
								    ||
								   /  \
								  /    \
					 ...   Faculty		Faculty  ...
			 _____________/  / \
		   /  				/   \
		  /   ...  Departament  Departament ...
	   Groups   		|
		 |			 Teachers
	  Students
  
</pre>

Система дозволяє керувати і редагувати користувачів, студентів, вчителів та складових університету: кафедри, факультети та групи. 
Розроблені ролі користувачів, адмін, менеджер та юзер мають різні рівні доступу та захищені бітовими масками. 
На рівні університету доступні повні звіти про: університет, учнів, вчителів. 
Реалізовано додавання факультетів та сортування студентів конкретного факультету за курсами.
Доступний рівень студента з усіма операціями керування, переводу та пошуку. Також реалізовано механізм додавання та редагування оцінок, розрахунок середнього балу.

Система побудована на взаємодії клієнт-сервер.
Клієнт відправляє запит на сервер, який обровлюється контролером та виконує сервіс. 
Далі відповідь відправляє сервер, з пакуванням даних у байт масив.
Клієнт отримує цю відповідь, хендлер який відповідає за це нормалізує до потрібного йому типу даних та виводить у консоль отриману відповідь у вигляду списку, або строки.

Конкретні методи захищені анотаціями @Secured, сервіси помічено @Service.

Бітові маски доступу допомагають захистити та взнати, чи має конкретний юзер відповідні права на виконання операції.
Ролі адмін, менеджер, та юзер мають різні набори.
Адмін має повний функціонал, менеджер обмежений всім, корім змоги видаляти та додавати структури університету, в той час як юзер має право READ-ONLY.

Реалізовано механізм багатопоточності, коли сервер може обробляти до 10 клієнтів одночасно. 
Кожен клієнт-запит має свій пріорітет завдяки synchronized.
Сервер завжди активний, чекаючи на клієнта для підключення, в той час як клієнт може від'єднатися, і сервер самостійно захендлить це.
